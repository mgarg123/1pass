# Security Policy

## Supported Versions

Currently, only the latest release of 1Pass is actively supported with security updates.

## Reporting a Vulnerability

Security and privacy are the core of 1Pass. If you discover a security vulnerability in the application (especially involving the cryptography, storage, or authentication layers), we ask that you **do not disclose it publicly**. Public disclosure before a fix is available could put existing users at risk.

Instead, please report it privately:

1. Send an email to the repository maintainer (or open a GitHub Security Advisory privately if available on the repository).
2. Include steps to reproduce the vulnerability.
3. Include any relevant logs, error messages, or potential attack vectors.

We will acknowledge your report as quickly as possible, and work with you to understand the issue, patch it, and release an update before public disclosure.

Thank you for helping keep 1Pass secure!
